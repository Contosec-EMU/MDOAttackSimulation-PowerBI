# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly by [opening a private security advisory](../../security/advisories/new) on this repository.

**Please do NOT create public GitHub issues for security vulnerabilities.**

### What to Include

- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any relevant logs or screenshots (ensure no sensitive data is included)

### Response Timeline

- **Initial response**: Best effort, typically within a few days
- **Resolution**: Dependent on severity; critical issues are prioritized

### Supported Versions

Security patches are provided for the latest version only. Please ensure you are using the most recent release before reporting.

## Contact

For security-related inquiries, use the [private security advisory](../../security/advisories/new) feature. For general questions, open a standard GitHub issue.
